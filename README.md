# 🇺🇸 English

## What is this?

AI Engineering Platform is an open engineering platform designed to orchestrate the complete software development lifecycle using Artificial Intelligence.

Unlike AI coding assistants that focus on generating code, this platform focuses on **engineering**.

The goal is to help software engineers design, build, validate, test, document and deploy software while maintaining engineering quality, consistency and traceability.

The generated software is **not** the product.

**The engineering platform is the product.**

---

## What this project is NOT

This project is **NOT**:

- A website generator.
- A CMS.
- A low-code platform.
- A no-code platform.
- A prompt collection.
- A wrapper around an AI model.

---

## Vision

Create an AI-native engineering platform capable of orchestrating the complete Software Development Life Cycle (SDLC) while keeping engineers in control of every important decision.

---

## Current Status

🚧 Product Design & Architecture Phase

No production code has been written yet.

The current focus is defining the product, architecture, engineering principles and platform foundation.

# 🇲🇽 Español

## ¿Qué es?

AI Engineering Platform es una plataforma de ingeniería diseñada para orquestar todo el ciclo de vida del desarrollo de software mediante Inteligencia Artificial.

A diferencia de otras herramientas enfocadas únicamente en generar código, esta plataforma busca asistir en todo el proceso de ingeniería.

Su objetivo es ayudar a diseñar, construir, validar, probar, documentar y desplegar software manteniendo altos estándares de calidad, consistencia y trazabilidad.

El software generado **no es el producto**.

**La plataforma de ingeniería es el producto.**

---

## Lo que este proyecto NO es

Este proyecto **NO** es:

- Un generador de sitios web.
- Un CMS.
- Una plataforma Low-Code.
- Una plataforma No-Code.
- Una colección de prompts.
- Un simple wrapper sobre un modelo de IA.

---

## Visión

Construir una plataforma de ingeniería basada en IA capaz de orquestar todo el ciclo de vida del desarrollo de software, manteniendo siempre al ingeniero como responsable de las decisiones importantes.
